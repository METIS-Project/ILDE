Assessment of participation through
Interaction analysis
Problem:

Participation in collaborative work affects each student's learning. However, under certain conditions it will be impossible to observe directly the collaborative processes; this may be caused by a high number of students, or in distance-learning settings.

Precisely, distance work, with computer support, can be especially difficult to observe and monitor.

Solution:

In those cases where collaboration is mediated by computer support, it is possible to gather valuable data about collaborative processes by means of interaction analysis software tools.

This requires:

  • The collaborative activity to be assessed must be carried out through a computer system (such as group work software).
  • This group work software must be capable of producing logs, which record user actions and thus the interactions between students through the system.
  • The assessor will use these logs to understand the collaborative process.
  • This is possible by using a second system, or interaction analysis system, capable of processing the logs to infer information about the behavior of the students.
  • Finally, it is necessary to make sure that both systems share the same data and format for the logs, and to connect them so that the logs produced by one system can be processed by the second.

This assessment can be carried out continuously or, at least, can be conducted several times along the collaborative activities.

Once the logs have been processed and analyzed, the results can be used with different purposes: give feedback to the students about their own work, take decisions about future learning activities, etc.

Application guidelines:

In order to include this assessment technique, is is recommended:

  • This pattern defines the assessment technique used. Therefore, it cannot be combined with any other pattern that also defines an assessment technique. However, patterns that propose some other additional configuration can be combined with this one.
  • In the assessed learning activity, collaboration must mediated by some group work (or collaborative learning) software system capable of producing logs. For example, logs are generated by systems such as BSCW or Synergeia. If these systems are used, they should appear as a resource available for the students in the assessed learning activity.
  • In the assess task, the analysis requires an interaction analysis tool, such as "i-bee", "COLEMON", "Observer XT", etc. This system should be included as a resource for the assessor in this activity, along with instructions for the assessor as to how to use it.
  • An example of an especialized interaction analysis is Role detection.
